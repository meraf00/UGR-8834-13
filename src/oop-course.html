Object Oriented Programming Concepts

Encapsulation

Encapsulation is a process of wrapping code and data together into a single unit. Encapsulation is the idea of:

  • Bundling data and function (logic) together
  • Exposing an interface whereby other objects can access and use it
  • Restricting access to certain details

Abstraction

The client of a class cares about the functionality the class offers, but not about how the functionality is implemented. This is referred to as data abstraction. Although programmers may know the details of a class's implementation, they should not write code that depends on these details. This enables a class to be replaced with another version without affecting the rest of the system.

Inheritance

Inheritance is the ability of classes to inherit attributes and methods from other classes. Generalization principle of OOP allows us to consider general categories of objects which have common properties. It allows the programmer to build a relationship between a new class and an existing class and define a new code in terms of existing code. In inheritance relationship there are two types of classes:

  • Superclass/Parent class : The class from where a subclass inherits features is called superclass. It is also called base class or parent class in java.
  • Subclass/Child class : A class that inherits all the members (attributes and methods) from other class is called subclass. It is also called a derived class, child class, or extended class.

Polymorphism

Polymorphism means that similar objects can respond to the same message in different ways. For example, you might have a system with many shapes. However, a circle, a square, and a star are each drawn differently. Using polymorphism, you can send each of these shapes the same message (for example, Draw), and each shape is responsible for drawing itself.
